Architecture: Two Components, One Seamless Experience

[Browser UI] ──── HTTP POST ────► [FastAPI Backend] │ ┌──────────┴──────────┐ [ChromaDB] [Ollama LLM] RAG Retrieval Response Gen. └──────────┬──────────┘ │ ┌──────────┴──────────┐ [Edge TTS] [Rhubarb] Speech Audio Lip-Sync Cues └──────────┬──────────┘ │ JSON { text, expression, animation, audio } │ [3D Avatar Renderer] ◄──── Three.js ◄──── React Components
  • Frontend: React 18 + Three.js WebGL canvas served via Vite
  • Backend: Python FastAPI handling AI, audio, and lip-sync in one call
  • Communication: single REST endpoint — text in, structured JSON + audio out
  • All computation is local — no external API calls at runtime
Speaker Notes The system has two layers. The browser renders the 3D avatar using WebGL and React Three Fiber. The Python backend handles all the intelligence — retrieving context, generating the response, converting it to speech, and computing lip-sync timings. The two communicate via a single REST endpoint.

Frontend Render
  • RAG: Query embedded and matched against ChromaDB — retrieves the 5 most relevant document chunks
  • LLM: Local Ollama model generates structured JSON: { text, facialExpression, animation }
  • TTS: Microsoft Edge TTS converts text to neural speech — no API key required
  • Lip-Sync: Rhubarb performs phonetic analysis → per-frame mouth cue timings (visemes)
  • Result: perfectly synchronized audio + visuals from a single backend call
Speaker Notes A user's question gets embedded into a vector, matched against your documents in ChromaDB, and the top 5 most relevant passages are injected into the LLM's prompt. The model returns not just the text answer, but which facial expression and animation to use. Then Edge TTS speaks it, and Rhubarb maps every phoneme to a mouth shape.

The Avatar: Lifelike Movement and Expression

🎭
9 Animation States
Idle, Talking (×3 variants), Laughing, Angry, Crying, Rumba, Terrified — with smooth 0.5s cross-fade transitions
😊
7 Facial Expressions
Default, Smile, FunnyFace, Sad, Surprised, Angry, Crazy — mapped across 27+ morph targets via lerp interpolation
👄
Frame-Accurate Lip-Sync
Viseme morph targets (viseme_AA, viseme_PP, etc.) timed to audio playback — every syllable has the correct mouth shape
👁️
Natural Eye Animation
Randomized blinking at 1–5 second intervals plus wink support for personality and realism
Speaker Notes The avatar isn't just a talking head. It cycles through three talking animations to avoid looking robotic, smoothly transitions between states, and its face uses 27 blend shapes to form expressions. The lip movements are phoneme-accurate — every syllable maps to the correct mouth shape, synchronized frame by frame with the audio.
